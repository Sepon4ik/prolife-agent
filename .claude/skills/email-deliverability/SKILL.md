# Email Deliverability for Cold Outreach

## When to use
When building email sending infrastructure, configuring domains, implementing warmup, or debugging deliverability issues.

## Domain Infrastructure

### Domain Strategy
Never send cold emails from your primary domain. Buy separate domains:

```
Primary: prolife.swiss (never use for cold outreach)

Cold outreach domains (buy 3-5):
  getprolife.com
  prolife-global.com  
  tryprolife.com
  prolife-partners.com
  prolifehealth.com
```

**Naming rules:**
- Include your brand name (recognizable, not spammy)
- Use .com, .io, .co (not .xyz, .biz, .info)
- Avoid hyphens if possible
- $10-15/domain/year

### DNS Configuration (per domain)

**SPF Record** — specifies who can send email for your domain:
```
Type: TXT
Name: @
Value: v=spf1 include:_spf.google.com include:amazonses.com include:resend.com ~all
```

**DKIM** — cryptographic signature proving email authenticity:
- Generated by your email provider (Google Workspace, Resend, etc.)
- Add the CNAME or TXT records they provide
- Usually 2 CNAME records per provider

**DMARC** — tells receivers what to do with unauthenticated email:
```
Type: TXT
Name: _dmarc
Value: v=DMARC1; p=none; rua=mailto:dmarc@prolife.swiss
```
Start with `p=none` (monitor only). After 2 weeks with clean data, move to `p=quarantine`, then `p=reject`.

**Custom tracking domain** (for open/click tracking):
```
Type: CNAME
Name: track
Value: track.resend.com (or your ESP's tracking domain)
```
**Critical:** Never use the ESP's default tracking domain — it's shared and may be blacklisted.

### Email Accounts per Domain
- 2-3 accounts per domain (e.g., outreach@, partnerships@, hello@)
- Use Google Workspace ($6/mo/account) or Microsoft 365 ($6/mo)
- Each account: unique display name, unique signature

## Email Warmup

### What warmup does
Gradually builds sender reputation by exchanging real emails with a warmup network. Mailbox providers (Gmail, Outlook) see consistent sending patterns and positive engagement (opens, replies, not-spam).

### Warmup Schedule

| Week | Emails/day | Notes |
|------|-----------|-------|
| Week 1 | 5-10 | All warmup, no cold emails |
| Week 2 | 15-25 | All warmup |
| Week 3 | 25-35 | Start sending 5-10 cold emails |
| Week 4 | 35-50 | Cold: 15-20/day + warmup continues |
| Week 5+ | 40-50 | Cold: 30-40/day + 10-15 warmup (maintenance) |

### Warmup Services
- **Instantly Warmup** — included with Instantly plan, largest warmup network
- **Warmup Inbox** — standalone, $15/mo per account
- **Lemwarm** — included with Lemlist

**Key rule:** Never stop warmup completely. Keep 10-15 warmup emails/day even during active campaigns.

## Sending Best Practices

### Volume & Timing
- **Max per account:** 30-50 cold emails/day (lower is safer)
- **Sending window:** 8am-5pm in recipient's timezone
- **Spread sends:** Random delays of 30-90 seconds between emails
- **Days:** Tuesday-Thursday are best. Avoid Monday AM and Friday PM.
- **Ramp gradually:** Don't jump from 0 to 50. Add 5-10/day.

### Content Rules for Inbox Placement

**DO:**
- Plain text (no HTML formatting)
- Short emails (50-125 words)
- One link max (your meeting booking link)
- Personalized opening line
- Clear unsubscribe method (reply "stop" or footer link)
- Questions in subject line

**DON'T:**
- HTML templates with images
- Multiple links
- Attachments
- ALL CAPS or excessive punctuation (!!!)
- Spam trigger words: "free", "guaranteed", "limited time", "act now", "click here"
- Same exact email to everyone (use spintax/variants)

### Spintax for Unique Emails
```
Hi {firstName|there},

{I noticed|I saw|I came across} {your company|{companyName}} 
{while researching|when looking at} {industry} {distributors|partners} 
in {country}.

{We help|Our team at ProLife helps|ProLife specializes in helping} 
{companies like yours|distributors in your market|partners in {country}} 
{expand their portfolio|grow with|add} Swiss medical technology products.

{Would you be open to|Could we schedule|I'd love to have} 
a {quick|brief|15-minute} {call|chat|conversation} this week?
```

## Mailbox Rotation

### How it works
```
Lead 1 → outreach@getprolife.com
Lead 2 → partnerships@prolife-global.com  
Lead 3 → hello@tryprolife.com
Lead 4 → outreach@getprolife.com (rotate back)
...
```

### Implementation pattern (for ProLife Agent)
```typescript
// Round-robin mailbox selection
const mailboxes = await prisma.mailbox.findMany({ 
  where: { isActive: true, dailySentCount: { lt: 40 } },
  orderBy: { dailySentCount: "asc" },
});
const selectedMailbox = mailboxes[0]; // Least-used today
```

### Domain Health Rotation
- Monitor delivery rate per domain weekly
- If a domain drops below 90% delivery → pause for 2 weeks
- Retire domains after 6-12 months and replace with fresh ones
- Keep 2-3 domains in warmup as reserves

## Monitoring & Diagnostics

### Key Tools
- **Google Postmaster Tools** — see domain reputation, spam rate, authentication
- **MXToolbox** — check blacklists, DNS, SMTP
- **mail-tester.com** — send test email, get deliverability score
- **GlockApps** — inbox placement testing across providers

### Alert Thresholds
```
Bounce rate > 5% → PAUSE immediately, clean list
Spam complaints > 0.3% → PAUSE, review content
Delivery rate < 90% → Check DNS, warmup, content
Open rate < 20% → Check deliverability (not just copy)
```

### Weekly Health Check
1. Check Google Postmaster for spam rate and domain reputation
2. Run mail-tester.com test from each sending account
3. Check MXToolbox for blacklists
4. Review bounce logs and remove bad addresses
5. Check warmup engagement ratios
